Core Algorithm

Weighted Multi-Factor Scoring

Required Skill Coverage70%
Optional Skill Bonus20%
Prerequisite Chain Credit10%

Prerequisite Chains: A student who knows JavaScript gets 35% partial credit towards React — because JavaScript is a prerequisite for React. Knowing a parent skill isn't ignored.

// Score = Required (70) + Optional (20) + Prereq (10) const requiredScore = (matched + prereqCredit) / total * 70; const optionalScore = matchedOptional / totalOptional * 20; const prereqBonus = skillsWithPrereq / missingCount * 10; const readinessScore = Math.min(100, Math.round( requiredScore + optionalScore + prereqBonus ));

Challenges & Learnings

Key Engineering Decisions

⚠ Challenges Faced
  • Skill aliases — "ML" ≠ "Machine Learning" caused false mismatches
  • Avoiding stale data without hammering external APIs on every request
  • CORS configuration for dev (Vite 5173) vs production (Vercel)
  • Preventing duplicate DB records on repeated seeding
✔ How We Solved Them
  • Built skillAliases.js with 100+ normalisation mappings
  • Implemented a 6-hour MongoDB TTL cache with background refresh
  • Dynamic origin allowlist in server.js; Vite proxy for local development
  • Used findOneAndUpdate with upsert:true keyed on applyUrl
📚 Design Insights
  • Simple % match is unfair — prerequisite chains add nuance
  • Optional skills must be weighted less than required skills
  • Real-time UI feedback > download a report (usability lesson)
🚀 Future Scope
  • ML-based skill recommendation engine
  • Resume parser to auto-extract skills from PDF
  • Recruiter dashboard for posting opportunities
  • Email alerts when new matching roles appear
